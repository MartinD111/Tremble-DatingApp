### Bug Fix — Tab Scroll State Reset

**Symptom:** Navigating away from Settings/Profile tab and returning reset the scroll position to the top.

**Root cause:** `AnimatedSwitcher` + `KeyedSubtree(key: ValueKey<int>(navIndex))` in `home_screen.dart` destroyed and recreated every screen widget on every tab switch. The `wantKeepAlive` mixin in `SettingsScreen` was ineffective — it only applies inside `PageView`/`TabBarView`, not `AnimatedSwitcher`.

**Fix:** Replaced `AnimatedSwitcher` with `IndexedStack` (commit `0511bf1`). All tab screens now stay alive in the widget tree; switching tabs changes only the visible index. Scroll position preserved.

**Trade-off:** Tab transition animation removed. Can be re-added with per-child `AnimatedOpacity` if desired.
