# Plan 11-02: Firestore proximity_events Write Rule Hardening

## Objective
Replace the permissive `allow write: if request.auth != null` rule on proximity_events with a strict rule that validates ownership, required fields, and field types.

## Tasks

<task id="T1" title="Replace proximity_events write rule in firestore.rules">
  <read_first>
    - firestore.rules (full file — locate the proximity_events match block)
  </read_first>

  <action>
    In firestore.rules, find the `match /proximity_events/{eventId}` block.
    It currently has a simple write rule: `allow write: if request.auth != null;`

    Replace the ENTIRE match block (keep the same match path) with:

    ```
    match /proximity_events/{eventId} {
      allow write: if request.auth != null
        && request.resource.data.from == request.auth.uid
        && request.resource.data.keys().hasAll(['from', 'toDeviceId', 'rssi', 'timestamp', 'ttl'])
        && request.resource.data.rssi is int
        && request.resource.data.timestamp is timestamp;
      allow read: if false;
    }
    ```

    Important:
    - Preserve all surrounding comments and other match blocks exactly as-is
    - The match path stays identical: `match /proximity_events/{eventId}`
    - The global deny rule at the bottom (`match /{document=**} { allow read, write: if false; }`) must remain untouched and as the last rule
    - Do NOT modify any other match blocks
  </action>

  <acceptance_criteria>
    - `grep -A8 "proximity_events" firestore.rules` shows the 6-line write rule with all conditions
    - `grep "request.resource.data.from == request.auth.uid" firestore.rules` returns exactly 1 match
    - `grep "hasAll" firestore.rules` returns 1 match containing 'from', 'toDeviceId', 'rssi', 'timestamp', 'ttl'
    - `grep "rssi is int" firestore.rules` returns exactly 1 match
    - `grep "timestamp is timestamp" firestore.rules` returns exactly 1 match
    - `grep -A2 "proximity_events" firestore.rules | grep "allow read: if false"` returns a match (read still denied)
    - `grep "allow read, write: if false" firestore.rules` still returns 1 match (global deny intact)
  </acceptance_criteria>
</task>
